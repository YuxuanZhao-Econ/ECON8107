Solving the HH problem

The following function (iterate_value) compues the value function and the policy function from the HH problem. It is smarter than brute force value function iteration as it exploits that the policy function is monotonic and the the problem is concave.

In[5]:
@nb.jit
def u(c):
    return c ** (1 - rho) / (1 - rho) 

@nb.jit
def iterate_value(v_0, bigR, ytilde_g, tolerance=10**(-4)):
    v_1 = np.empty((grid_points, n))
    pol_1 = np.empty((grid_points, n), dtype=np.int32)
    p_transpose = p.transpose()
    beta_v_exp = np.empty((grid_points, n))
    iter = 0
    while True:
        distance = 0.0
        iter += 1
        for s in range(n):  # pre-computing the expected value term
            for i in range(grid_points):
                accum = 0.0
                for sprime in range(n):
                    accum = accum + beta * p[s, sprime] * v_0[i, sprime]
                beta_v_exp[i, s] = accum
        for s in range(n):  # doing the optimization
            pol = 0
            for i in range(grid_points):
                just_started = True
                j = pol # policy is monotonic -- start from the previous optimal policy
                while j < grid_points:
                    c = bigR * ahat_g[i] + ytilde_g[s] - ahat_g[j]
                    if c > 0.0:
                        temp = u(c) + beta_v_exp[j, s]
                        if just_started:
                            vmax = temp
                            pol = j
                            just_started = False
                        elif temp > vmax:
                            vmax = temp
                            pol = j
                        else:
                            # value decreased -- concavity means it can't go up again
                            break
                    j += 1
                v_1[i, s] = vmax
                pol_1[i, s] = pol
                dis = abs(vmax - v_0[i, s])
                if dis > distance: 
                    distance = dis
        if distance < tolerance:
            break
        else:
            temp2 = v_0
            v_0 = v_1
            v_1 = temp2
    return v_1, pol_1

The stationary distribution

The following function takes an initial pdf and iterates using the policy function to obtain the stationary distribution

In[8]:
@nb.jit
def get_stationary_distribution(pdf_0, policy, tolerance=10**(-7)):
    while True:
        pdf_1 = np.zeros((grid_points, n))
        for i in range(grid_points):
            for s in range(n):
                for sprime in range(n):
                    pdf_1[policy[i, s], sprime] += p[s, sprime] * pdf_0[i, s]
        distance = 0.0
        for i in range(grid_points):
            for s in range(n):
                distance = max(abs(pdf_1[i, s] - pdf_0[i, s]), distance)
        if distance - tolerance < 0:
            break
        else:
            pdf_0 = pdf_1
    return pdf_1

(b) Numerical Method of Solving HH problem

The code solves a standard income-fluctuation (Aiyagari) household problem.

Let the state be $(a,s)$. The Bellman equation is $$ V(a,s)=\max \left\{ u(c)+\beta\sum_{s'\in S}p_{ss'}V(a',s') \right\}, $$ subject to $$ a'\ge -\phi \\ c+a' = Ra + y(s). $$

To remove the lower bound $a\ge -\phi$ from the choice set, define shifted variables $$ \hat a \equiv a+\phi $$ Then the constraints become $$ a\ge -\phi \iff \hat a\ge 0 $$

Therefore the original problem is equivalent to a new Bellman equation:

$$ V(\hat a,s) = \max \left\{ u\big(c\big) +\beta\sum_{s'\in S} p_{ss'}\,V(\hat a',s') \right\}, $$ where $$ \hat a' \geq 0 \\ c + \hat a'= R \hat a + \tilde y(s) \\ \tilde y(s)=y(s)-(R-1)\phi. $$

We use VFI to solve this HH problem

Each VFI step does two key things:

1. Precompute continuation values

For each current shock $s$ and each candidate next-asset index $j$, compute once: $$ \beta\,\mathbb E\!\left[V(\hat a_j,s')\mid s\right] =\beta\sum_{s'\in S}p_{ss'}\,V(\hat a_j,s'). $$ This avoids recomputing the expectation inside the inner maximization loop.

2. Maximize over $j$ efficiently (monotonicity + break on decline)

Fix a shock $s$ and a current grid point $\hat a_i$. The code maximizes over next-asset grid indices $j$ the discrete objective $$ \Psi_{i,s}(j) \equiv u\!\left(c_{i,s}(j)\right) +\beta\sum_{s'\in S}p_{ss'}\,V(\hat a_j,s'), \qquad c_{i,s}(j)=R\hat a_i+\tilde y(s)-\hat a_j, $$ subject to feasibility $c_{i,s}(j)\ge 0$ (equivalently $\hat a_j\le R\hat a_i+\tilde y(s)$).

Let $j^*(i,s)\in\arg\max_j \Psi_{i,s}(j)$.

Monotone search (start from previous maximizer).
A standard property in this problem is that the optimal saving policy is increasing in cash-on-hand / assets, so the index policy is nondecreasing: $$ j^*(i+1,s)\ \ge\ j^*(i,s). $$ Therefore, when moving from $\hat a_i$ to $\hat a_{i+1}$ (holding $s$ fixed), the code starts the search at $$ j_{\text{start}} = j^*(i,s), $$ instead of restarting from $j=0$.

Early stopping (break on decline).
As $j$ increases, current consumption $c_{i,s}(j)$ falls, while the continuation term rises at a diminishing rate; numerically $\Psi_{i,s}(j)$ is typically single-peaked in $j$ (unimodal). The code exploits this by stopping once the objective begins to fall: $$ \text{if }\Psi_{i,s}(j+1) < \Psi_{i,s}(j)\text{ at some }j,\ \text{then stop searching larger }j. $$ Under unimodality, once the objective decreases it will not exceed the current best again, so the maximizer has already been passed.

Given an optimal policy $j^*(i,s)$, the economy becomes a finite-state Markov chain over states $(i,s)$. If $\mu_t(i,s)$ is the cross-sectional distribution, the forward update is: $$ \mu_{t+1}(j^*(i,s),s') \;+=\; p_{ss'}\,\mu_t(i,s)\qquad\text{for all }s'\in S. $$ Iterating this push-forward map until convergence yields the stationary distribution $\mu$. Marginal distributions (PDF/CDF over assets) are obtained by summing over $s$: $$ \mu_{\hat a}(i)=\sum_{s\in S}\mu(i,s). $$

(c) Policy functions (assets and consumption) as functions of cash-in-hands

For each current employment state $s$, cash-in-hands is $z=R\hat a + y_s$. The code below plots $a'(z,s)$ and $c(z,s)$.

From the asset policy plot $a'(z,s)$, for a given level of cash-on-hand $z$ the household typically chooses higher savings when unemployed

Why?

Because the income process is persistent, Thus, at the same $z$, unemployment means lower current resources going forward and a higher chance of binding the borrowing constraint, so the household wants to save more (precautionary saving), shifting $a'(z,\text{unemployed})$ upward relative to $a'(z,\text{employed})$.

(e) Fraction borrowing constrained

With $\phi=0$, the borrowing constraint binds when the optimal choice is $a'(z,s)=0$ (the lowest grid point).

The fraction borrowing constrained is the stationary mass at the lower bound $a'=0$: $$ \Pr(a'=0)=\sum_i\sum_{s}\mu(i,s)\,\mathbf 1\{a'(z_i,s)=0\}. $$ In the code this comes out to about $1.8\%$

This is lower than I expected. My interpretation is: in this incomplete-markets environment, agents have a precautionary saving motive (which is even stronger when people have lower income and higher chance of be binded). Being at $a'=0$ is costly because it removes self-insurance exactly when future income risk is still there, so households try to stay away from the cliff by holding a strictly positive buffer whenever possible.

(f) Stationary distribution under lower risk aversion (log utility)

Set $\rho=1$ and resolve the household problem, then recompute the stationary distribution and compare moments/plots.

In this model, $\rho$ controls risk aversion. When $\rho$ decreases, the agent has less incentive to smooth consumption across states.

Formally, with CRRA $u(c)=\frac{c^{1-\rho}}{1-\rho}$, a lower $\rho$ means utility is less curved, so fluctuations in $c$ are less painful.

Key implication:

  • The utility loss from hitting the borrowing constraint (choosing the lowest feasible $a'$) is smaller when $\rho$ is smaller.
  • Therefore the precautionary saving motive weakens, so agents are less willing to accumulate buffers to avoid the constraint.

So the stationary distribution shifts toward lower assets, and the constrained fraction increases: $$ \rho \downarrow \quad\Rightarrow\quad \text{precautionary saving } \downarrow \quad\Rightarrow\quad \Pr\!\big(a'(z,s)=a_{\min}\big)\uparrow, $$ i.e. more mass at the borrowing limit and a higher fraction borrowing constrained under $\rho=1$.

# --- Helper: solve the HH problem + stationary distribution for a given rho ---
def solve_hh(rho_val, bigR_val, w_val, grid_points_val=2000, ahat_max_val=15.0, tol_v=1e-4, tol_pdf=1e-7):
    # grids
    ahat_g_val = np.linspace(0.0, ahat_max_val, grid_points_val)
    y_g_val = w_val * l_g
    ytilde_g_val = y_g_val - (bigR_val - 1) * phi

    # utility
    if rho_val == 1:
        def u_local(c):
            return np.log(c)
    else:
        def u_local(c):
            return c ** (1 - rho_val) / (1 - rho_val)

    # value iteration (same monotone/concave search idea, but in pure python for flexibility)
    v0 = np.zeros((grid_points_val, n))
    v1 = np.empty_like(v0)
    pol = np.empty((grid_points_val, n), dtype=np.int32)
    beta_v_exp = np.empty_like(v0)

    while True:
        # expected value term
        for s in range(n):
            for i in range(grid_points_val):
                beta_v_exp[i,s] = beta * np.dot(p[s,:], v0[i,:])
        dist = 0.0
        for s in range(n):
            pol_s = 0
            for i in range(grid_points_val):
                j = pol_s
                started = False
                while j < grid_points_val:
                    c = bigR_val * ahat_g_val[i] + ytilde_g_val[s] - ahat_g_val[j]
                    if c > 0:
                        val = u_local(c) + beta_v_exp[j,s]
                        if not started:
                            vmax = val
                            pol_s = j
                            started = True
                        elif val > vmax:
                            vmax = val
                            pol_s = j
                        else:
                            break
                    j += 1
                v1[i,s] = vmax
                pol[i,s] = pol_s
                dist = max(dist, abs(v1[i,s]-v0[i,s]))
        if dist < tol_v:
            break
        v0, v1 = v1, v0

    # stationary distribution
    pdf0 = np.ones((grid_points_val, n)) / (grid_points_val * n)
    while True:
        pdf1 = np.zeros_like(pdf0)
        for i in range(grid_points_val):
            for s in range(n):
                for sp in range(n):
                    pdf1[pol[i,s], sp] += p[s,sp] * pdf0[i,s]
        if np.max(np.abs(pdf1-pdf0)) < tol_pdf:
            break
        pdf0 = pdf1

    Abar_val = np.sum(ahat_g_val[:,None] * pdf1)
    constrained_val = np.sum(pdf1[pol==0])
    L_val = np.sum(l_g[None,:] * pdf1)  # mean labor supply
    return ahat_g_val, ytilde_g_val, v1, pol, pdf1, Abar_val, constrained_val, L_val

(h) Firm-side mappings $W(R)$ and $K(R)$

With CobbDouglas $F=A K^\alpha L^{1-\alpha}$ and depreciation $\delta$, competitive pricing implies $$R = 1 + \alpha A (K/L)^{\alpha-1} - \delta,\qquad w=(1-\alpha)A (K/L)^\alpha.$$ Use $w(1.04)=1$ to pin down $A$.

alpha = 0.36
delta = 0.1

def k_over_l_from_R(R, A_prod):
    return (alpha * A_prod / (R - 1 + delta))**(1/(1-alpha))

def W_of_R(R, A_prod):
    x = k_over_l_from_R(R, A_prod)
    return (1-alpha) * A_prod * (x**alpha)

def K_of_R(R, A_prod, Lbar_val):
    x = k_over_l_from_R(R, A_prod)
    return x * Lbar_val
